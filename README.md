# Music Lyrics AI Studio

Современный локальный проект для **генерации музыки по описанию, тексту и аудио-примеру** с интерфейсом на **Gradio** и developer-режимом для подключения по API.

> Важное ограничение: в этом репозитории используется **готовая предобученная модель MusicGen от Meta / Hugging Face**, которая скачивается при первом запуске. Это уже реальная модель генерации аудио, но я не могу честно обещать, что она «не уступает всем популярным сервисам» — качество зависит от железа, длины генерации, промпта и выбранного пресета.

---

## Что умеет проект

- **Генерация по текстовому описанию**: настроение, жанр, инструменты, структура, атмосфера.
- **Генерация по тексту песни / вокальной идее**: lyrics добавляются в итоговый conditioning prompt.
- **Генерация по аудио-примеру / мелодии**: можно загрузить свой референс и использовать его как ориентир.
- **Gradio-приложение**: проект запускается как обычный локальный сайт в браузере.
- **Developer menu**: внутри интерфейса можно создать API-ключ и сразу получить готовый `curl` пример.
- **REST API**: можно поднять отдельный HTTP API и подключить генератор к своему сайту, боту или приложению.
- **Локальное сохранение артефактов**: каждая генерация сохраняет `.wav` и `.json` с параметрами.

## Какие модели поддерживаются

Через аргумент `--preset` можно выбрать один из готовых чекпоинтов:

- `small` → `facebook/musicgen-small`
- `medium` → `facebook/musicgen-medium`
- `melody` → `facebook/musicgen-melody` **(рекомендуется для большинства случаев)**
- `large` → `facebook/musicgen-large`

### Что выбрать

- **Если ноутбук слабый / без видеокарты** → `small`
- **Если нужен лучший стартовый вариант** → `melody`
- **Если у вас мощный GPU** → `medium` или `large`

Если не знаете, что выбрать — берите **`melody`**.


#### macOS / Linux

```bash
source .venv/bin/activate
```

Когда окружение включилось, в начале строки терминала обычно появляется что-то вроде:

```text

```

```bash
lyrics-ai launch --preset melody --host 127.0.0.1 --port 7860
```

   - Safari
3. Вставьте адрес в адресную строку браузера.

## Как пользоваться интерфейсом: куда нажимать внутри приложения

После открытия страницы вы увидите вкладки:

- **Studio**
- **Developer**

Сначала работаем во вкладке **Studio**.

### Вкладка `Studio`: пошагово

#### 1. Поле `Описание трека`

Сюда нужно **написать, какую музыку вы хотите**.

Например:

```text
epic dark trailer with hybrid drums, huge bass, female vocal textures and cinematic rise
```

#### 2. Поле `Стиль`

Нажмите на выпадающий список и выберите стиль, например:

- `cinematic`
- `synthwave`
- `rock`
- `ambient`
- `trap`

Если не знаете, что брать — оставьте `cinematic`.

#### 3. Поле `Текст / вокальная идея`

Это необязательно.

Сюда можно вставить, например, строчку будущей песни:

```text
We rise above the fire tonight
```

Если вокал не нужен — оставьте поле пустым.

#### 4. Поле `Референсы и комментарии`

Сюда можно написать пояснение:

- как должен начинаться трек;
- когда должен быть дроп;
- какое настроение должно быть в финале.

Например:

```text
Slow atmospheric intro, then strong impact around 8 seconds, ending should feel massive and heroic
```

#### 5. Поле `Аудио-пример / мелодия`

Если хотите, чтобы модель ориентировалась на ваш пример:

1. Нажмите на блок загрузки аудио.
2. Выберите `.wav` / `.mp3` файл с компьютера.
3. Дождитесь, пока файл прикрепится.

Если пример не нужен — ничего не загружайте.

#### 6. Правые настройки

Справа будут слайдеры и поля.

- **Длительность (сек.)** — сколько секунд генерировать
- **Guidance scale** — насколько сильно модель старается следовать описанию
- **Temperature** — насколько результат будет более смелым / хаотичным
- **Top-k** и **Top-p** — параметры сэмплирования
- **Seed** — число для повторяемости результата

Для первого запуска можно оставить всё как есть.

#### 7. Кнопка `Показать итоговый промпт`

Нажмите эту кнопку, если хотите сначала посмотреть, какой итоговый текст уйдёт в модель.

После нажатия ниже появится поле:

- **Промпт, ушедший в модель**

#### 8. Кнопка `Сгенерировать музыку`

Когда всё заполнено:

1. Нажмите **Сгенерировать музыку**.
2. Подождите.
3. На слабом ноутбуке ожидание может быть долгим.
4. После завершения появится:
   - **плеер** с результатом,
   - **итоговый промпт**,
   - **метаданные генерации** в формате JSON.

#### 9. Как послушать результат

После генерации в блоке **Результат**:

1. Нажмите кнопку **Play**.
2. Прослушайте сгенерированный аудиофайл.
3. Если результат не понравился:
   - поменяйте описание,
   - поменяйте стиль,
   - измените `seed`,
   - снова нажмите **Сгенерировать музыку**.

После этого ваш API будет доступен по адресу:

```text
http://127.0.0.1:8000
```

## Частые проблемы и решения

### Проблема: команда `python` не найдена

Попробуйте:

```bash
python3 --version
```

Если `python3` работает, используйте его вместо `python` в командах.

### Проблема: долго скачивается модель

Это нормально при первом запуске.

Что делать:

- подождать;
- проверить интернет;
- не закрывать терминал;
- попробовать сначала более лёгкий пресет `small`.

### Проблема: в PowerShell видно `^C`, а до этого как будто всё зависло

Если у вас в терминале было что-то вроде:

```text
(.venv) PS C:\\Users\\...> ^C
```

это значит, что вы **сами прервали процесс через `Ctrl+C`**.

Чаще всего это происходит в двух ситуациях:

1. вы ждёте первый запуск и модель ещё скачивается;
2. идёт первая генерация, а на CPU она выполняется долго.

Что делать правильно:

- не нажимать `Ctrl+C`, если терминал просто долго думает на первом запуске;
- дождаться сообщений вида `[music-lyrics-ai] Loading pretrained model ...`;
- дождаться завершения скачивания модели;
- если ноутбук слабый, запускать так:

```bash
lyrics-ai launch --preset small --device cpu
```

Если вы уже нажали `Ctrl+C`, просто запустите команду ещё раз.

### Проблема: генерация очень медленная

Это нормально на CPU.

Что можно сделать:

- использовать `--preset small`
- уменьшить длительность
- закрыть другие тяжёлые программы
- если есть GPU — явно указать `--device cuda`

### Проблема: не открывается сайт `127.0.0.1:7860`

Проверьте:

1. Терминал всё ещё открыт?
2. В терминале нет ошибки?
3. Вы точно запускали `launch`, а не `serve-api`?
4. Вы открываете именно `http://127.0.0.1:7860`, а не другой адрес?

### Проблема: PowerShell блокирует активацию

Выполните:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

А потом снова:

```powershell
.\.venv\Scripts\Activate.ps1
```

### Проблема: результат плохой

Попробуйте:

- сделать описание более конкретным;
- сменить `style`;
- добавить `reference_notes`;
- поменять `seed`;
- использовать `melody` вместо `small`.

## Архитектура проекта

- **`music_lyrics_ai/service.py`** — собирает итоговый промпт, нормализует параметры, загружает MusicGen, запускает инференс и сохраняет `.wav`/`.json`.
- **`music_lyrics_ai/gradio_app.py`** — интерфейс на Gradio с вкладками `Studio` и `Developer`.
- **`music_lyrics_ai/api.py`** — FastAPI-приложение с endpoint'ами `/health`, `/v1/generate`, `/v1/files/{filename}`.
- **`music_lyrics_ai/api_keys.py`** — локальное хранилище API-ключей.
- **`music_lyrics_ai/config.py`** — пресеты моделей и dataclass-конфиги.
- **`music_lyrics_ai/cli.py`** — команды `launch`, `serve-api`, `create-key`.

---

## Где сохраняются результаты

По умолчанию:

- аудио: `artifacts/renders/*.wav`
- метаданные: `artifacts/renders/*.json`
- API-ключи: `artifacts/api_keys.json`

## Практические советы по качеству

- Пишите промпт как brief для композитора или саунд-дизайнера.
- Для лучшего контроля добавляйте структуру: `intro`, `build`, `drop`, `outro`.
- Для мелодического переноса загружайте чистый короткий аудио-пример.
- Если результат слишком случайный — уменьшите `temperature`.
- Если модель слабо слушается описания — увеличьте `guidance_scale`.
- Если хотите сравнить варианты — меняйте только `seed`.

## Ограничения

- MusicGen ограничен примерно **30 секундами общего контекста** за один прогон.
- `large` и иногда `medium` могут быть слишком тяжёлыми для слабых ноутбуков.
- На CPU генерация может идти долго.
- API-ключи здесь локальные и простые — это удобная developer-функция, а не enterprise IAM.

## Почему это лучше предыдущей версии репозитория

Предыдущая версия была учебной символьной сетью для **генерации текста песен**, а не музыки. Теперь проект переключён на **реальную генерацию аудио** через современную предобученную модель, UI-приложение и developer API, то есть соответствует задаче намного ближе.
