# Music Lyrics AI Studio

Современный локальный проект для **генерации музыки по описанию, тексту и аудио-примеру** с интерфейсом на **Gradio** и developer-режимом для подключения по API.

> Важное ограничение: в этом репозитории теперь используется **готовая предобученная модель MusicGen от Meta / Hugging Face**, которая скачивается при первом запуске. Это реальный современный генератор музыки, но я не могу честно назвать его «лучше всех популярных коммерческих сервисов» — качество зависит от железа, длины генерации, промпта и выбранного чекпоинта.

## Что теперь умеет проект

- **Генерация по текстовому описанию**: mood, жанр, инструменты, структура, темп, атмосфера.
- **Генерация по тексту песни / вокальной идее**: lyrics добавляются в conditioning prompt.
- **Генерация по аудио-примеру / мелодии**: можно загрузить wav/mp3 и использовать как ориентир для продолжения или ремикса.
- **Gradio-приложение**: интерфейс, который открывается как обычное локальное веб-приложение.
- **Developer menu**: внутри UI можно создать API-ключ и получить готовый `curl` пример.
- **REST API**: можно поднять отдельный HTTP API и подключать генератор из другого приложения.
- **Локальное сохранение артефактов**: каждый рендер сохраняет `.wav` + `.json` с параметрами запуска.

## Архитектура

- **`music_lyrics_ai/service.py`** — главный orchestration layer: собирает итоговый промпт, нормализует параметры, вызывает backend генерации и сохраняет `.wav`/`.json`.
- **`music_lyrics_ai/gradio_app.py`** — UI на Gradio с вкладками `Studio` и `Developer`.
- **`music_lyrics_ai/api.py`** — FastAPI-приложение с endpoint'ами `/health`, `/v1/generate`, `/v1/files/{filename}`.
- **`music_lyrics_ai/api_keys.py`** — локальное хранилище API-ключей: ключи создаются как plaintext, а сохраняются в виде hash.
- **`music_lyrics_ai/config.py`** — пресеты моделей и dataclass-конфиги приложения.
- **`music_lyrics_ai/cli.py`** — CLI-команды для запуска UI, API и создания ключей.

## Какие модели поддерживаются

Через аргумент `--preset` можно выбрать один из готовых чекпоинтов:

- `small` → `facebook/musicgen-small`
- `medium` → `facebook/musicgen-medium`
- `melody` → `facebook/musicgen-melody` **(рекомендуется)**
- `large` → `facebook/musicgen-large`

### Что выбрать

- **Для ноутбука без мощного GPU**: `small`
- **Для баланса качество / скорость**: `melody`
- **Для сильной машины с GPU**: `medium` или `large`

## Что нужно для локального запуска

Минимально:

1. **Python 3.10+**
2. **Git** (желательно, но не строго обязательно)
3. **Интернет при первом запуске**, чтобы скачать предобученную модель
4. **Достаточно места на диске** под модель и генерации
5. Желательно **GPU**, но CPU тоже поддерживается

## Установка

### 1. Создайте виртуальное окружение

#### Windows PowerShell

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

#### Windows CMD

```cmd
python -m venv .venv
.venv\Scripts\activate.bat
```

#### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Установите зависимости

Полная установка для приложения, API и инференса:

```bash
python -m pip install -U pip
python -m pip install -e .[app,inference,dev]
```

Если нужен только код и тесты без тяжёлой модели:

```bash
python -m pip install -e .[dev]
```

## Быстрый запуск Gradio-приложения

```bash
lyrics-ai launch --preset melody --host 127.0.0.1 --port 7860
```

После запуска откройте браузер:

```text
http://127.0.0.1:7860
```

### Как пользоваться интерфейсом

#### Вкладка `Studio`

1. В поле **Описание трека** напишите, какую музыку хотите.
   - пример: `epic dark trailer with hybrid percussion, distorted bass and female vocal textures`
2. В выпадающем списке **Стиль** выберите жанр / направление.
3. При желании вставьте **Текст / вокальную идею**.
4. В **Референсы и комментарии** можно уточнить структуру трека, настроение, драматургию.
5. В **Аудио-пример / мелодия** можно загрузить свой референс.
6. Нажмите **Показать итоговый промпт**, чтобы увидеть final conditioning prompt.
7. Нажмите **Сгенерировать музыку**.
8. После генерации появятся:
   - плеер с `.wav`,
   - итоговый промпт,
   - JSON с метаданными.

#### Вкладка `Developer`

1. Введите имя клиента, например `desktop-client`.
2. Нажмите **Создать API-ключ**.
3. Скопируйте ключ и сохраните в безопасном месте.
4. Ниже появится готовый `curl` пример.
5. Затем поднимите API отдельной командой:

```bash
lyrics-ai serve-api --preset melody --host 127.0.0.1 --port 8000
```

## Пример запуска REST API

Сначала создайте ключ:

```bash
lyrics-ai create-key --label desktop-client
```

Потом поднимите API:

```bash
lyrics-ai serve-api --preset melody --host 127.0.0.1 --port 8000
```

Проверка здоровья сервиса:

```bash
curl http://127.0.0.1:8000/health
```

Пример генерации:

```bash
curl -X POST http://127.0.0.1:8000/v1/generate \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: ВАШ_КЛЮЧ' \
  -d '{
    "description": "dark cyberpunk trailer with huge bass and glitch drums",
    "style": "synthwave",
    "lyrics": "We run through neon fire tonight",
    "duration_seconds": 10,
    "seed": 42
  }'
```

API вернёт путь к готовому `.wav`, prompt и ссылку на скачивание файла.

## Где сохраняются результаты

По умолчанию:

- аудио: `artifacts/renders/*.wav`
- метаданные: `artifacts/renders/*.json`
- API-ключи: `artifacts/api_keys.json`

## Практические советы по качеству

- Для **лучшего результата по описанию** старайтесь писать промпт как brief для саунд-дизайнера.
- Для **более управляемой генерации** добавляйте структуру: intro, build, drop, outro.
- Для **мелодического переноса** загружайте короткий чистый аудио-пример.
- Для **повторяемости** фиксируйте `seed`.
- Если результат слишком хаотичный — уменьшите `temperature`.
- Если модель слишком слабо слушается промпта — поднимите `guidance_scale`.

## Ограничения

- MusicGen ограничен примерно **30 секундами общего контекста** за один прогон.
- `large` и иногда `medium` могут быть слишком тяжёлыми для слабых ноутбуков.
- На CPU генерация может идти долго.
- API-ключи здесь локальные и простые — это удобный developer feature, а не enterprise IAM.

## Почему это лучше предыдущей версии репозитория

Предыдущая версия была учебной символьной сетью для **генерации текста песен**, а не музыки. Теперь проект переключён на **реальную генерацию аудио** через современную предобученную модель, UI-приложение и developer API, то есть соответствует задаче намного ближе.
